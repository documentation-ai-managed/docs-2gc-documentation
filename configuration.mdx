## Environment Variables

Use environment variables to configure 2GC without modifying configuration files directly. This is especially useful for containerized deployments.

| Variable | Description | Default |
|----------|-------------|---------|
| `DOCS_PORT` | Server port | `3000` |
| `DOCS_HOST` | Server host | `localhost` |
| `DOCS_DB_URL` | Database URL | Required |

<Expandable title="Database Configuration" default-open="false">
  <Expandable title="SQLite" default-open="false">
    For development: `DOCS_DB_URL="sqlite:./docs.db"`
  </Expandable>
  <Expandable title="PostgreSQL" default-open="false">
    For production: `DOCS_DB_URL="postgresql://user:pass@host:port/db"`

<Request tabs="Validate Config" show-lines="false">
  ```javascript
  const validateConfig = async (config) => {
    const response = await fetch('/api/config/validate', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(config)
    });
    const result = await response.json();
    return result.valid;
  };
  ```
</Request>
